## Анализ

На любых данных гибридная версия выигрывает у стандартной, рассмотрим разные сценарии:
#### Случайные массивы (random):

n=10000:

Гибридная: 2758 мс

Стандартная: 4286 мс

#### Почти отсортированные массивы (almost_sort):

n=50000:

Гибридная: 11267 мс

Стандартная: 19296 мс

#### Обратно отсортированные массивы (sorted_rev):

Стандартная: 40590 мс

## Итоговый вывод
Гибридный merge sort с применением insertion sort для малых отрезков демонстрирует стабильно лучшую производительность по сравнению со стандартным merge sort, особенно на почти отсортированных и случайных данных.
